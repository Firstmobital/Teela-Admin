## What Was Configured

### 1. PWA Support (vite-plugin-pwa)

✅ **Service Worker (sw.js)** — Auto-generated by vite-plugin-pwa
- Pre-caches app shell (HTML, CSS, JS)
- Caches menu items and activities for offline viewing
- Does NOT cache chat, orders, payments, bill (real-time data)
- Automatically updates when new code is deployed

✅ **Web Manifest (manifest.webmanifest)** — Auto-generated from vite.config.js
```json
{
  "name": "Teela Resort",
  "short_name": "Teela",
  "theme_color": "#3D1F08",
  "background_color": "#F9F6F1",
  "display": "standalone",
  "start_url": "/",
  "scope": "/"
}
```

✅ **Service Worker Registration** — Auto-injected via registerSW.js
- Automatically registers on page load
- Updates check runs every hour
- Safe auto-update mechanism

✅ **Caching Strategy**
- **Menu items:** CacheFirst (24h expiry) → Fast load, auto-updates daily
- **Activities:** CacheFirst (24h expiry) → Static content safe to cache
- **Reservations:** NetworkFirst (1h cache) → Prefers fresh data, offline fallback
- **Chat/Orders/SOS:** Not cached → Always fresh (real-time required)

### 2. Vercel Configuration (vercel.json)

✅ **SPA Routing Configured**
```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```
- All routes redirect to index.html (React Router handles them)
- `/staff/login`, `/stay/verify`, `/stay/:token/home` all work correctly

## Ready-to-Deploy Checklist

Before deploying to Vercel:

- [ ] **1. Generate PWA Icons** (see PWA_ICONS_SETUP.md)
  - Create/place `public/icon-192.png`
  - Create/place `public/icon-512.png`
  - Create/place `public/icon-512-maskable.png`

- [ ] **2. Set Up Supabase Backend** (see SUPABASE_SETUP.md)
  - Create Supabase project
  - Create database tables (migrations)
  - Enable RLS policies
  - Enable Realtime publications
  - Deploy Edge Functions (checkin, preferences, activity, order, folio, payment, payment/verify, sos)
  - Set Edge Function secrets

- [ ] **3. Test Locally**
  - `npm install` (if needed)
  - `npm run dev`
  - Test check-in flow: `/stay/verify`
  - Test staff portal: `/staff/login`
  - Test offline: DevTools > Network > Offline
  - Verify service worker: DevTools > Application > Service Workers

- [ ] **4. Connect to Vercel**
  - Push code to GitHub main branch
  - Go to vercel.com, import repository
  - Wait for auto-deployment

- [ ] **5. Set Vercel Environment Variables**
  - `VITE_SUPABASE_URL`
  - `VITE_SUPABASE_ANON_KEY`
  - `VITE_RAZORPAY_KEY_ID`
  - Redeploy to apply variables

- [ ] **6. Test Production** (at vercel URL)
  - Full check-in to payment flow
  - Staff operations
  - PWA installation

- [ ] **7. Configure Custom Domain** (optional)
  - Add domain in Vercel
  - Update DNS CNAME record
  - Wait for SSL certificate
